# Information Security Exam Questions 2026

A shared repository for collecting answers to the possible exam questions for the *Information Security* course (Eric Laermans – Dirk Deschrijver), academic year 2025–2026.

The full list of questions and cases is taken from `examquestions-2026-final.pdf` (2026-05-06).

## Structure

- `cases/` — one markdown file per security case (`case-01.md` … `case-37.md`).
- `questions/` — one markdown file per "other" question (`question-01.md` … `question-54.md`).
- `examquestions-2026-final.pdf` — the original question list from the lecturers.

Each file contains the question text followed by an `## Answer` section. Unanswered files have `TODO` as a placeholder.

## This repo is for everyone

This is a collective effort. **Everyone is expected to contribute.** Studying for the exam is a lot easier if we split the work — pick a case or question that hasn't been answered yet (or one where you think the existing answer can be improved) and fill it in.

A few ground rules:

- Don't be afraid to edit or improve someone else's answer. Discussion and corrections are welcome.
- If you disagree with an existing answer, add your reasoning rather than silently overwriting it.
- Keep answers focused on the question — quote the relevant slides/notes where useful.

## When you push an answer, mention your source

It's very helpful for the rest of us to know **how** you arrived at an answer, so we can judge how much to trust it and where to dig deeper. Please mention your source(s) at the bottom of the answer, for example:

- AI (which model, e.g. ChatGPT, Claude, Gemini, …)
- Course notes / slides only
- Answers from previous years
- The textbook / external reference (link or citation)
- Your own reasoning
- A combination of the above

A short note like *"Source: course slides chapter 4 + sanity-checked with Claude"* is plenty. This makes it easier to spot answers that need a second pair of eyes versus those that have already been verified against the course material.

Good luck and happy studying!
